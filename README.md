# JellyInstall

A self-hosted movie and TV show downloader with a modern web UI. Browse and search TMDB, then download directly to your server using a fast multi-connection downloader.

## Features

- Browse trending movies and TV shows with a hero carousel
- Search TMDB for any movie or TV show
- Download movies and TV shows with a 20-connection range-request downloader
- TV shows are saved as `<show_dir>/<Show Name>/Season XX/<filename>`
- JWT-based authentication with admin user management
- Light/dark mode (Space Grotesk font, black and white theme)
- Static Next.js frontend served by FastAPI — no separate web server required

## Requirements

- Python 3.11+
- Node.js 18+ and npm (for building the frontend)

## Quick Start (Development)

```bash
# 1. Install Python dependencies
pip install -r requirements.txt

# 2. Build the frontend
python build.py

# 3. Create an admin user
python -m backend.app.manage create-user --username admin --password yourpassword --admin

# 4. Start the server
python run.py
```

The app will be available at `http://localhost:8097`.

## Configuration

All configuration is done via environment variables:

| Variable | Default | Description |
|---|---|---|
| `PORT` | `8097` | Port to listen on |
| `JWT_SECRET` | random (regenerated on restart) | Set a fixed value in production |
| `MOVIE_DIR` | `~/Movies` | Directory to save downloaded movies |
| `SHOW_DIR` | `~/Shows` | Directory to save downloaded TV shows |

## Linux Production Install (systemd)

Use the interactive install script to install JellyInstall as a systemd service:

```bash
chmod +x install.sh
sudo ./install.sh
```

The script will:
1. Check for required dependencies (Python 3.11+, Node.js 18+, git)
2. Ask for: service user, port, movie/show download directories, admin username/password
3. Set up a Python virtual environment
4. Build the frontend
5. Write configuration to `/etc/jellyinstall/env`
6. Create and enable a systemd service (`jellyinstall.service`)
7. Create the admin user and start the service

After installation:
- Service: `systemctl status jellyinstall`
- Logs: `journalctl -u jellyinstall -f`
- Config: `/etc/jellyinstall/env`

## User Management

```bash
# Create a user
python -m backend.app.manage create-user --username <name> --password <pass>

# Create an admin user
python -m backend.app.manage create-user --username <name> --password <pass> --admin

# List all users
python -m backend.app.manage list-users
```

## Project Structure

```
jellyinstall1/
  backend/
    app/
      main.py          # FastAPI app, static serving, SPA catch-all
      config.py        # Configuration (reads env vars)
      auth.py          # JWT + bcrypt authentication
      database.py      # SQLite via async SQLAlchemy
      models.py        # User, Settings, DownloadRecord
      manage.py        # CLI user management
      routers/         # auth, tmdb, download, admin
      services/        # tmdb_service, download_engine
    static/            # Built frontend (generated by build.py)
  frontend/
    app/               # Next.js app router pages and components
  build.py             # Builds frontend and copies to backend/static/
  run.py               # Starts the uvicorn server
  install.sh           # Interactive Linux systemd install script
  requirements.txt
```
